# Prototype Scope - AI Wedding Producer
*מסמך מטרה לדמו הצוות - שבוע מ-6.5.2025*

## מה בונים

פרוטוטייפ של מהלך שלם מא׳ עד ת׳, צר ועמוק:
משתמש חדש → הבנת החתונה → בחירת 3 ספקים → דשבורד מלא.

## המהלך המלא

### שלב 1 - Onboarding (דרך שיחה, לא טופס)
הסוכן אוסף בקצב טבעי:
- שמות הזוג
- תאריך (ספציפי / חודש / טווח)
- מספר אורחים
- תקציב כולל
- סוג טקס (רבנות / רפורמי / אזרחי / destination)
- סגנון / וייב (מודרני / קלאסי / בוטיק / כפרי)
- אזור גיאוגרפי

### שלב 2 - 3 ספקים פעילים
לאחר onboarding הסוכן מוביל לבחירת:
1. **אולם / מקום** (venue)
2. **קייטרינג**
3. **צילום סטילס** (photography)

לכל ספק - הסוכן:
- שואל שאלות מצמצמות (קיבולת, סגנון, אזור, תקציב)
- מציג 2-3 אופציות כ-כרטיסי ספק בצ'אט
- עוקב אחרי התקדמות (collecting → negotiating → hold → signed)
- מעדכן הדשבורד בזמן אמת

### שלב 3 - דשבורד חי
כל מה שנאסף בשיחה מופיע בדשבורד:
- Hero widget: שמות + תאריך + countdown
- 3 סלוטים פעילים עם סטטוס צבעוני
- Budget widget: תקציב כולל vs. הוצאות צפויות

## מה לא בונים (דחוי)
- ספקים נוספים מעבר ל-3 (שמלה, רב, מוזיקה וכו׳)
- מסך ספק מלא עם גלריה
- push notifications
- multi-user / שיתוף משפחה
- מרקטפלייס ספקים

## הגדרת הצלחה
משתמש שפותח את האפליקציה בפעם הראשונה יכול להגיע בשיחה טבעית אחת לדשבורד עם:
- שמות + תאריך + אורחים + תקציב מלאים
- לפחות ספק אחד במצב negotiating או מעלה
- הדשבורד משקף את כל מה שנאמר בשיחה
